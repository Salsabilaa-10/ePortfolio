Labs

Lab 1 — User Interface (UI)

Building foundational UI layouts using Jetpack Compose.

Rebuilding the student planner home screen with Jetpack Compose was a great learning experience, though nesting the layout components was a bit tricky at first. I struggled with managing the custom padding and sizes to ensure the elements fit neatly without overlapping. By experimenting with the layout modifiers and practicing structural nesting, I managed to build a clean and responsive user interface.

Lab 2 — Interaction

Handling user events, state management, and interactive components.

Implementing state management in Lab 2 was highly rewarding, though it was tricky at first when the UI didn't update after clicking the send button because I missed how recomposition is triggered. I also struggled with making the calendar section horizontally scrollable to allow users to swipe through more dates neatly. By studying the layout documentation and experimenting with mutableStateOf and the horizontalScroll modifier, I successfully resolved both issues on my own.

Lab 3 — Material Design

Applying Material Design 3 components for a polished, accessible UI.

Upgrading StudyMate to Material Design in Lab 3 was a very rewarding experience, though figuring out the exact theme color roles for components like primaryContainer was a bit tricky. I also struggled at first with configuring the spring settings to get a smooth, natural bounce animation for the expandable cards. By practicing layout adjustments and checking the documentation for the spring constants, I successfully styled the theme and created the interactive card animations myself.

Lab 4 — Navigation

Implementing multi-screen navigation with Jetpack Navigation Component.

Implementing proper navigation and state management with a NavController and StudyMateViewModel was a highly rewarding experience in Lab 4. It was quite tricky at first to set up the NavHost routes and ensure the user data from the UserData data class shared perfectly between the Home and Profile screens. By practicing state collection and adjusting the animateDpAsState offset parameters, I successfully created a smooth, multi-screen navigation flow with an interactive animated nav bar on my own.

Lab 5 — Room Database

Persisting data locally using the Room ORM library.

Integrating the AddTaskScreen and AddExamScreen with input validation was a highly rewarding experience in Lab 5. It was a bit tricky at first to handle the state updates so that newly added items would instantly display back on the Home and History screens via the shared StudyMateViewModel. By practicing form validation logic and ensuring proper data flows through our repository layer, I successfully managed to make the data persistence work smoothly across all screens on my own.
